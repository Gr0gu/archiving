# SUDO Archiver (`.sudo`)

A custom, hybrid lossless data compression and archiving tool written entirely in C. Developed by Team SUDO (Group FAF-252) for the Algorithm Analysis and Implementation course.

## Overview
The SUDO archiver combines file bundling with a multi-stage hybrid compression pipeline. It borrows concepts from both `bzip2` and `DEFLATE` to achieve high compression ratios. It takes multiple files, applies advanced transformations, and outputs a single `.sudo` archive. 

Because general file archiving requires perfect data recovery, all algorithms implemented are strictly lossless.

## The Compression Pipeline
When a user creates a `.sudo` archive, the data flows through the following stages, each implemented natively:

1. **Archiver/Container:** Bundles files and directories, prepends metadata (filenames, original sizes, permissions), and calculates CRC-32 checksums for integrity.
2. **Burrows-Wheeler Transform (BWT):** Reorganizes the input data to group similar characters together.
3. **Move-to-Front (MTF):** Transforms the BWT output into lower-value integers, optimizing it for the next stages.
4. **Run-Length Encoding (RLE):** Compresses the long runs of identical values (mostly zeros) generated by the MTF transform.
5. **Huffman Coding:** The final entropy coding stage that builds a frequency-based MinHeap to assign optimal variable-length binary codes to the final byte stream.

*Decompression runs this exact pipeline in reverse, utilizing the header metadata to verify file integrity via checksums.*

## Compression Levels

The archiver supports five compression levels (chosen with the `-l` flag):

| Level | Pipeline                        | Chunk Size | Speed  | Ratio  |
|-------|---------------------------------|------------|--------|--------|
| 1     | Huffman                         | ~50 KB     | Fastest | Lowest |
| 2     | RLE → Huffman                   | ~100 KB    | Fast    | Low    |
| 3     | MTF → RLE → Huffman             | ~200 KB    | Medium  | Medium |
| 4     | BWT → MTF → RLE → Huffman       | ~400 KB    | Slow    | High   |
| **5** | **BWT → MTF → RLE → Huffman**   | **~800 KB**| **Slowest** | **Best** |

Level **4** is the default and matches the behavior of the original archiver. Level 5 uses the same full pipeline but processes larger chunks, giving the BWT more context to improve compression ratios on large files.

## Password Encryption

Archives can be optionally protected with a password using the `-p` flag. The encryption uses an XOR stream cipher (the password is applied cyclically as the key) and is applied to the compressed data after each pipeline stage, before writing to disk. On extraction, decryption is applied before decompression.

> **Note:** XOR encryption provides basic obfuscation. It is not intended to replace cryptographically strong encryption.

## Build & Run

### Prerequisites
* GCC or Clang compiler
* Make

### Compilation
```bash
make all```

### Usage

**Compress** one or more files into an archive:
```bash
./sudo_archiver -c [options] <archive.sudo> <file1> [file2 ...]
```

**Extract** all files from an archive:
```bash
./sudo_archiver -x [options] <archive.sudo>
```

### Options

| Flag              | Description                                             |
|-------------------|---------------------------------------------------------|
| `-l <1-5>`        | Compression level (default: `4`)                        |
| `-p <password>`   | Encrypt (compress) or decrypt (extract) with password   |

### Examples

```bash
# Compress with default settings (level 4)
./sudo_archiver -c archive.sudo file1.txt file2.txt

# Compress at level 1 (fastest)
./sudo_archiver -c -l 1 archive.sudo bigfile.bin

# Compress at level 5 (best ratio) with password protection
./sudo_archiver -c -l 5 -p mysecret archive.sudo report.pdf

# Extract a plain archive
./sudo_archiver -x archive.sudo

# Extract a password-protected archive
./sudo_archiver -x -p mysecret archive.sudo
```
