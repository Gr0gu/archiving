# SUDO Archiver (`.sudo`)

A custom, hybrid lossless data compression and archiving tool written entirely in C. Developed by Team SUDO (Group FAF-252) for the Algorithm Analysis and Implementation course.

## Overview
The SUDO archiver combines file bundling with a multi-stage hybrid compression pipeline. It borrows concepts from both `bzip2` and `DEFLATE` to achieve high compression ratios. It takes multiple files, applies advanced transformations, and outputs a single `.sudo` archive. 

Because general file archiving requires perfect data recovery, all algorithms implemented are strictly lossless.

## The Compression Pipeline
When a user creates a `.sudo` archive, the data flows through the following 6 stages, each implemented natively:

1. **Archiver/Container:** Bundles files and directories, prepends metadata (filenames, original sizes, permissions), and calculates CRC-32 checksums for integrity.
2. **Burrows-Wheeler Transform (BWT):** Reorganizes the input data to group similar characters together.
3. **Move-to-Front (MTF):** Transforms the BWT output into lower-value integers, optimizing it for the next stages.
4. **Run-Length Encoding (RLE):** Compresses the long runs of identical values (mostly zeros) generated by the MTF transform.
5. **LZ77:** Applies a sliding-window dictionary approach to catch and reference remaining larger repeating patterns.
6. **Huffman Coding:** The final entropy coding stage that builds a frequency-based MinHeap to assign optimal variable-length binary codes to the final byte stream.

*Decompression runs this exact pipeline in reverse, utilizing the header metadata to verify file integrity via checksums.*
